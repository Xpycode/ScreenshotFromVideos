### Research (parallel)
Dispatched 2 parallel general-purpose agents with /ultrathink-level briefs:

**Agent 1 — SDWebImageWebPCoder recon:**
- v0.15.0 (Nov 2025), actively maintained, ICC profile embedding added
- SPM URL `https://github.com/SDWebImage/SDWebImageWebPCoder.git`, single product `SDWebImageWebPCoder`
- Pulls SDWebImage core + libwebp-Xcode transitively → ~1.2–1.8 MB + ~0.6–0.9 MB after DCE
- Encode-only alternative: depend directly on `libwebp-Xcode` (SPM product `libwebp`) for ~50 LOC of Swift bridging, drops SDWebImage core
- Encode API: `SDImageWebPCoder.shared.encodedData(with: nsImage, format: .webP, options: opts)`. Lossless via raw key `SDImageCoderEncodeWebPLossless: NSNumber(value: true)`
- Swift 6 strict concurrency: needs `@preconcurrency import` on both SDWebImage + SDWebImageWebPCoder modules — headers not Sendable-audited
- Universal binary (arm64+x86_64) works; source-built; no xcframework
- Notarization + hardened runtime clean — pure source, no dlopen, no JIT, no `cs.disable-library-validation`
- macOS 26 / Tahoe compatible (0.15.0 shipped Nov 2025 post-Tahoe Sept 2025)
- **Open issue #116 is load-bearing**: default `method=4` produces visibly lower output than `cwebp` CLI at same `q`. Mitigation: send `SDImageCoderEncodeWebPMethod: NSNumber(value: 6)` explicitly per encode.
- **Color space gotcha**: `AVAssetImageGenerator` returns CGImages tagged with video color space (Rec.709/BT.2020). libwebp writes pixels as-is. Convert via transient sRGB `CGContext` before NSImage handoff; 0.15.0 will then embed the resulting sRGB ICC profile.

**Agent 2 — libwebp encoding knobs + UX patterns:**
- libwebp's `WebPConfig` fields: `lossless`, `quality` (reused as effort in lossless), `method` (0–6), `image_hint`, `alpha_quality`/`alpha_compression`/`alpha_filtering`, `near_lossless`, `exact`, `use_sharp_yuv`, plus advanced (filter strength/sharpness, sns, segments, partitions, autofilter — skip for UI)
- Lossless quality semantic: q=100 → slow + smallest file; q=0 → fast + larger. Pixel data always bit-exact in lossless mode regardless of q.
- Real-app survey:
  - **Squoosh** (Google's ref): Lossless toggle, Quality, Effort, + Advanced panel (hint, alpha quality, sns, sharp YUV, exact)
  - **GIMP**: Lossless checkbox, Image quality, Alpha quality, Image-hint dropdown, Sharp YUV
  - **Photoshop (WebPShop)**: Lossless/Lossy radio + Quality + Fastest/Default/Slowest effort
  - **Affinity Photo**: Quality slider + Lossless checkbox only — most minimal
  - **macOS Preview**: cannot export WebP (read-only)
- **Recommendation (Affinity-minimal)**: Quality slider (reused for effort in lossless) + Lossless toggle. Honest to format's bimodal decision + continuous dial. Skip image_hint, method, alpha knobs.

## Decisions

(All routed into the plan; will be promoted to `docs/decisions.md` after WebP ships.)

- **WebP support via SDWebImageWebPCoder 0.15.0** — accepting ~2 MB binary growth in exchange for upstream maintenance and ICC-embed feature. Direct `libwebp-Xcode` wrapper logged as future-shrink option if size becomes an issue.
- **Two UI controls: Quality slider + Lossless toggle.** Affinity-minimal. Honest to the format's one bimodal decision + one continuous dial. Skipped: image_hint, method slider, alpha knobs, sharp_yuv, near_lossless — defaults are fine for short video stills.
- **Quality slider relabels to "Effort" when WebP + lossless is on.** libwebp reuses the `quality` param as encoding effort when `lossless=1`. One slider, two semantic roles. `.help()` tooltip explains the dual semantic.
- **Hardcode `SDImageCoderEncodeWebPMethod: NSNumber(value: 6)`** on every encode. Closes the SDWebImageWebPCoder #116 quality gap vs `cwebp` CLI. CPU cost 2–3× acceptable for batch.
- **Convert CGImage to sRGB via transient `CGContext`** before handing to the encoder. Otherwise Rec.709-tagged video frames render with shifted colors in color-unmanaged viewers. 0.15.0 will embed the resulting sRGB ICC profile.
- **`@preconcurrency import SDWebImageWebPCoder` + `@preconcurrency import SDWebImage`** needed under `SWIFT_STRICT_CONCURRENCY=complete`. Upstream headers not Sendable-audited.
- **Ship on a fresh `polish/webp-support` branch**, not stacked on `polish/post-phase-5`.
