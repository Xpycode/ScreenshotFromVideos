## Progress

### Strip / transport improvements — branch `polish/strip-transport` (4 atomic commits, off `polish/webp-support` HEAD)
Build green at every step; app relaunched + user-smoked after each change.

- **`e38d3ae` fix(strip): real zoom +/- buttons + keyboard focus retention.**
  The `minus.magnifyingglass` / `plus.magnifyingglass` icons flanking the zoom
  slider were bare `Image` views with no action — only the slider + reset were
  interactive. Wrapped them in `Button`s calling `model.zoomOut()`/`zoomIn()`.
  Clicking any zoom button stole keyboard focus from the strip, so subsequent
  `+`/`-` key presses died. Fix: `@FocusState private var stripFocused` bound via
  `.focused($stripFocused)`, buttons marked `.focusable(false)`, and
  `stripFocused = true` re-asserted in each button action.
- **`50b42af` feat(strip): arrow-key navigation.** ←/→ step the active cell ±1,
  ↑/↓ step ±`cellsPerRow` (tracked live via a second `onScrollGeometryChange(for: Int.self)`).
  Each arrow seeks the player to that cell's exact time (sample-accurate) and
  recenters via the existing `seekScrollTarget` path. New `navigate(by:)` anchors
  off `activeIndex` (player-derived) so it stays correct across other seeks.
  Tapping a cell now also sets `stripFocused = true` so arrows work after a click.
  **Gotcha:** macOS arrow keys carry implicit `.function`/`.numericPad` modifier
  flags, so an initial `press.modifiers.isEmpty` guard rejected *every* arrow —
  switched to the cmd/opt/ctrl-only guard the other handlers use.
- **`12bcb08` feat(strip): transport bar (timecode + frame # + zoom scale).**
  Always-visible bar at the top of the strip (Penumbra `ControlsRow` styling):
  play/pause toggle · selected frame as `HH:MM:SS:FF` + absolute `frame N` ·
  zoom-scale readout `every 20 fr · 0.66s` collapsing to `every frame · 1:1` at
  max. New `TimestampFormatter.timecode(from:fps:)` + `.frameNumber(from:fps:)`;
  `StripModel.framesPerThumbnail` / `secondsPerThumbnail`. Play icon tracks
  `AVPlayer.timeControlStatus` via a KVO async sequence (`publisher(for:).values`)
  so it's correct when playback ends on its own. Frame number rides a hidden
  worst-case-width template (cookbook #67) so the right scale doesn't jitter.
  User chose: top-of-strip placement, TC + frame #, play/pause only (no
  step/jump buttons — AVPlayerView's inline controls cover gross transport).
- **`fd351f6` fix(strip): lift density cap to source fps.** `density` capped at
  `min(nominalFPS, 30)`, so 60 fps clips topped out at every-2nd-frame and could
  never reach 1:1. Changed cap to `Double(nominalFPS)` (restores the documented
  Phase-5 intent). User confirmed `every frame · 1:1` at max zoom on a 60 fps clip.

### Export polish — branch `polish/export-extras` (2 atomic commits, off `polish/strip-transport` HEAD)

- **`65d23e5` fix(overlay): burn-in font as a fraction of frame height.**
  User: "burn timestamp might not work." It *did* work — proved it by vendoring
  `applyTextOverlay` into a throwaway Swift script (`/tmp/overlaytest.swift`) and
  rendering a PNG. Root cause: font was **absolute pixels** (16–96pt), so on a 4K
  frame it was ~1.7% of height — present but a corner speck. Replaced
  `OverlaySettings.fontSize` (CGFloat pt) with `fontPercent` (Double, default
  0.035), resolved to pixels per-frame in the pipeline (`CGImage.height * fontPercent`).
  Slider now 1–8% of height. **New `overlay.fontPercent` preference key** (NOT
  reusing `overlay.fontSize` — old `36` would read as 3600%). 4K verification:
  5754 white pixels vs old 1128, clearly legible.
- **`1f47c46` feat(export): Show in Finder button after export.** VM retains
  `lastExportedURLs` (set on success, cleared on new run) and exposes
  `revealInFinder()` (`NSWorkspace.activateFileViewerSelecting`). Button appears
  in the export footer in place of Cancel once a run completes. User chose
  "button after export" over auto-reveal.

## Decisions
- **Q = enqueue, ⌘C = copy-to-clipboard.** Bare `Q` stays on the strip's scoped
  `.onKeyPress` (can't shadow text-field typing — project convention); `⌘C` is a real
  button shortcut so a focused field still gets text-copy first.
- **Filename time tokens reuse the transport-bar formatters** so a name and the on-screen
  readout are guaranteed identical. `{frame}` zero-padded for correct lexical sort.
- **Default pattern now `{name}_{timecode}`** — traceable by default; stored patterns
  keep working.
- **Burn-in preview uses a computed aspect-fit rect, not `AVPlayerView.videoBounds`** —
  same geometry for `.resizeAspect`, avoids KVO/`Binding`-Sendable friction.
- **Merged the full polish stack to `main` via fast-forward**; no squash (atomic history
  preserved). Remote/push left as an open user decision.

- **Strip work on its own branch** (`polish/strip-transport`), export work on
  `polish/export-extras` — kept separable from WebP per user's branch-hygiene
  preference. Both stacked on the WebP branch (not main) because main is far
  behind: WebP + multiformat + post-phase-5 are all unmerged.
- **Transport bar = play/pause only.** AVPlayerView already has inline
  play/pause/scrub/frame-step (`controlsStyle = .inline`, `showsFrameSteppingButtons`),
  so the custom bar adds the always-visible TC + frame # + zoom scale, not a
  duplicate full transport.
- **Burn font is resolution-relative (% of height), not absolute pt** — the only
  sane choice across 1080p/4K. Migrated to a new preference key to avoid
  misreading old absolute values.
- **Density caps at source fps**, restoring documented intent; the 256-in-flight
  decode cap in `scheduleRender()` bounds the extra work on high-fps clips.
